name: legal-statute-search
description: 法律成文法/法规检索技能。当用户询问法条原文、法律规定、法律条款内容、某类行为的法律后果、具体法律名称中的条文时使用。仅限法律知识库 knowledge/legal/statutes/ 目录下检索。
---

## 目标

用法条原文、法律条款的准确定义回答用户问题。返回必须包含法条编号、法律名称、具体条文内容，必要时给出相关司法解释。

## 路由决策（必须先执行）

**规则**：分析用户问题，先输出路由决策 JSON，再调用任何工具。

| 优先级 | 条件 | 方法 |
|---|---|---|
| P0 | 含"A和B有什么不同"、"哪个更"、"对比" | Comparison Hop（跨法域/新旧法） |
| P1 | 含"为什么"、"怎么理解"、"司法解释怎么说" | Bridge Entity Hop（法条→司法解释） |
| P2 | 含"以前"、"现在"、"旧法"、"民法典实施前" | Temporal Chain Hop（新旧法对比） |
| P3 | 含"需要什么条件"、"构成要件"、"成立条件" | Legal Element Hop（要件拆解） |
| P4 | 以上都不匹配 | 单跳 |

```json
{"method":"Comparison Hop|Bridge Entity Hop|Temporal Chain Hop|Legal Element Hop|single","reason":"一句话说明","sub_queries":[{"step":1,"query":"...","tool":"kb_search|kb_metadata_filter"}]}
```

## 查询重写与意图挖掘

### 术语规范化映射表

| 口语化表达 | 专业术语 | 映射原因 |
|---|---|---|
| "违约了怎么办" | "违约责任 / 违约金 / 损害赔偿 / 继续履行" | 需映射到具体责任形式 |
| "不干了" / "不做生意了" | "合同解除 / 合同终止 / 退伙 / 解散" | 需区分不同法律后果 |
| "赔钱" | "损害赔偿 / 违约金 / 赔偿金 / 补偿金" | 不同性质的法律概念 |
| "打官司" | "诉讼 / 仲裁 / 调解 / 行政复议" | 纠纷解决方式不同 |
| "合同不生效" | "合同无效 / 合同可撤销 / 合同解除 / 合同终止 / 合同不成立" | 5种完全不同的法律后果 |
| "公司换老板" | "股权变更 / 法定代表人变更 / 公司合并 / 公司分立" | 需映射到具体法律行为 |
| "中介费" | "居间报酬 / 佣金 / 中介费 / 服务费" | 合同类型决定法律适用 |
| "离职后不能去竞争对手" | "竞业限制 / 竞业禁止 / 保密义务" | 需区分不同制度 |
| "买房后不想买了" | "商品房预售合同解除 / 买受人解除权 / 定金罚则" | 需映射到具体法律规则 |
| "欠钱不还" | "债务履行 / 迟延履行 / 根本违约 / 强制执行" | 需判断债务履行阶段 |

### 法条编号识别规则

**用户问题中可能隐含法条编号，需要提取：**

| 用户表达 | 提取的法条编号 | 说明 |
|---|---|---|
| "563条" | "民法典第563条" | 需结合上下文推断法律名称 |
| "合同法94条" | "合同法第94条" | 旧法，需提示民法典已取代 |
| "消法24条" | "消费者权益保护法第24条" | 简称映射 |
| "公司法第几条说注册资本" | "公司法 注册资本" | 无具体编号，需主题检索 |

### 意图挖掘检查清单

1. **用户要的是法条原文，还是要法条解释？**
   - "第563条怎么说的" → 要法条原文
   - "第563条是什么意思" → 法条原文 + 司法解释，可能需要 Bridge Entity Hop

2. **用户是否隐含了新旧法对比？**
   - "民法典关于合同解除的规定" → 新法，单跳
   - "合同法第94条现在还有效吗" → **新旧法对比，需要 Temporal Chain Hop**

3. **用户是否隐含了跨法域对比？**
   - "中国的竞业限制期限" → 单跳
   - "中国和美国的竞业限制有什么不同" → **Comparison Hop**

4. **用户是否把多个法律概念混在一起？**
   - "合同不生效了怎么办" → 需判断是"无效"、"可撤销"、"解除"还是"终止"，每个对应不同法条
   - "违约了可以解除合同吗" → **两个法律问题：违约责任 + 解除权，可能需要多跳**

5. **用户问的是行为合法性，还是法律后果？**
   - "跳单违法吗" → 合法性判断
   - "跳单要赔多少钱" → 法律后果量化，需要 Bridge Entity Hop（法条 → 损害赔偿计算）

### 查询重写策略

**策略1：法条编号优先**
```
原始："民法典563条"
重写："民法典 第五百六十三条 合同解除"（补全中文数字，加关键词）
```

**策略2：旧法→新法映射**
```
原始："合同法94条"
重写："合同法第94条 民法典第563条 合同解除 对应关系"
```

**策略3：行为→法条映射**
```
原始："欠钱不还怎么办"
重写："债务履行 迟延履行 违约责任 民法典"
```

**策略4：跨法域对比**
```
原始："中国和美国的竞业限制"
重写1："竞业限制 中国 劳动合同法"（中国法）
重写2："non-compete California employment law"（美国法）
```

## 多跳检索判断

### 单跳即可的场景

- 用户明确给出了法律名称 + 法条编号
- 用户问的是单一法律概念的定义
- 问题不涉及比较、新旧法、跨法域
- **示例**："民法典第563条怎么说的？"

### 需要多跳的场景

#### 场景A：Bridge Entity Hop（法条→司法解释）
**适用**：用户需要法条原文 + 司法解释的理解

**判断标准**：
- 问题含"是什么意思"、"怎么理解"、"司法解释怎么说"
- 问题涉及法条的具体适用条件

**示例**：
```
问题："民法典第563条的'合理期限'是多久？"
分析：需要法条原文 + 司法解释对"合理期限"的解释

多跳步骤：
  跳1：检索法条原文
    kb_search(query="民法典 第五百六十三条 合同解除", top_k=2)
    → 找到法条原文
    
  跳2：检索司法解释对"合理期限"的解释
    kb_search(query="合同解除 合理期限 司法解释", top_k=3)
    → 找到司法解释的具体规定
    
  融合：法条原文 + 司法解释 = 完整答案
```

#### 场景B：Temporal Chain Hop（新旧法对比）
**适用**：问题涉及新旧法律的对比或法律沿革

**判断标准**：
- 问题含"以前"、"现在"、"民法典实施前"、"旧法"
- 问题涉及法律修订、废止、替代

**示例**：
```
问题："民法典实施后，合同法第94条还有效吗？"
分析：需要对比旧法（合同法）和新法（民法典）

多跳步骤：
  跳1：检索旧法规定
    kb_search(query="合同法 第九十四条 合同解除", top_k=2)
    → 旧法条文
    
  跳2：检索新法规定
    kb_search(query="民法典 第五百六十三条 合同解除", top_k=2)
    → 新法条文
    
  跳3：检索法律适用过渡规定
    kb_search(query="民法典 时间效力 合同法 废止", top_k=2)
    → 最高人民法院关于民法典时间效力的司法解释
    
  融合：对比新旧法差异，说明现行有效规定
```

#### 场景C：Comparison Hop（跨法域对比）
**适用**：问题涉及不同法域的法律对比

**判断标准**：
- 问题含"中国和美国"、"国内和国外"、"不同国家"
- 问题涉及国际法律比较

**示例**：
```
问题："中国和美国的竞业限制期限分别是多少？"
分析：需要分别检索中国法和美国法，再对比

多跳步骤：
  跳1：检索中国法
    kb_search(query="竞业限制 期限 劳动合同法", top_k=2)
    → 中国法：最长2年
    
  跳2：检索美国法
    kb_search(query="non-compete duration California employment law", top_k=2)
    → 美国法：加州基本禁止，其他州差异大
    
  融合：对比两地法律差异
```

#### 场景D：Legal Element Hop（法律要件分析）
**适用**：问题涉及法律构成要件的分析

**判断标准**：
- 问题形式："X需要什么条件？"、"X的构成要件是什么？"
- 问题涉及法律行为的成立/生效条件

**示例**：
```
问题："合同解除需要什么条件？"
分析：需要拆解解除权的构成要件

多跳步骤：
  跳1：检索法定解除条件
    kb_search(query="民法典 第五百六十三条 合同解除 条件", top_k=2)
    → 要件1：不可抗力；要件2：明示违约；要件3：迟延履行；要件4：根本违约
    
  跳2：检索约定解除条件
    kb_search(query="民法典 第五百六十二条 约定解除 条件", top_k=2)
    → 约定解除权的行使条件
    
  跳3：检索解除程序要求
    kb_search(query="民法典 合同解除 通知 程序", top_k=2)
    → 解除权的行使程序
    
  融合：按法定解除、约定解除、程序要求组织答案
```

## 可用工具

| 工具名称 | 用途 | 必选参数 | 可选参数 |
|---|---|---|---|
| `kb_search` | 混合检索（向量 + BM25） | `query` | `top_k`, `path_filter` |
| `kb_metadata_filter` | 带元数据过滤的检索 | `query`, `filters` | `top_k` |
| `kb_open_chunk` | 打开指定 chunk 的完整内容 | `parent_id` | 无 |

---

## 检索变量说明

| 变量 | 含义 | 默认值 | 可选值 |
|---|---|---|---|
| `jurisdiction` | 法域/管辖地 | `"China"` | `"China"`, `"USA"`, `"EU"` |
| `law_name` | 法律名称 | 自动提取 | 如 `"Civil Code"`, `"Company Law"` |
| `article_number` | 法条编号 | 无 | 如 `"第563条"` |
| `year` | 法律修订年份 | 无 | 如 `2020`, `2021` |

## 执行步骤

### 单跳执行流程

1. **提取检索要素**
   - 法域（未明确则默认 China）
   - 法律名称（如有）
   - 法条编号（如有）
   - 核心法律术语（用术语映射表转换）

2. **查询重写**
   - 用"术语规范化映射表"转换口语化表达
   - 提取隐含的法条编号
   - 判断是否需要旧法→新法映射

3. **首次检索**
   - 有明确法条编号 → `kb_search`，query 含"法律名 + 编号 + 关键词"
   - 无法条编号 → `kb_metadata_filter`，filters 含 `document_type=statute`

4. **精确定位**
   - `kb_open_chunk` 打开 chunk 获取完整条文
   - 核对版本（优先最新生效版本）

### 多跳执行流程

1. **判断多跳类型**（见"多跳检索判断"章节）
2. **拆分子查询**
3. **逐跳执行**
4. **融合结果**

## 推荐命令

### 单跳：按法条编号检索

```python
kb_search(
    query="民法典 第563条 合同解除",
    path_filter="knowledge/legal/statutes/",
    top_k=3
)
```

### 多跳：法条→司法解释（Bridge Entity）

```python
# 跳1：法条原文
statute = kb_search(query="民法典 第五百六十三条 合同解除", top_k=2)

# 跳2：司法解释
interpretation = kb_search(query="合同解除 合理期限 司法解释", top_k=3)
```

### 多跳：新旧法对比（Temporal Chain）

```python
# 跳1：旧法
old_law = kb_search(query="合同法 第九十四条 合同解除", top_k=2)

# 跳2：新法
new_law = kb_search(query="民法典 第五百六十三条 合同解除", top_k=2)

# 跳3：过渡规定
transition = kb_search(query="民法典 时间效力 司法解释", top_k=2)
```

### 多跳：跨法域对比（Comparison）

```python
# 跳1：中国法
cn_law = kb_search(query="竞业限制 期限 劳动合同法", top_k=2)

# 跳2：美国法
us_law = kb_search(query="non-compete duration California", top_k=2)
```

## 结果筛选规则

- 优先使用**法条原文**
- 同一条款多个版本时，优先**最新生效版本**
- 如果结果中有"已废止"、"已被修订"等字样，必须重新检索当前有效版本
- 法条编号必须精确到条、款、项
- 跨法域问题必须标注法域来源

---

## 失败处理

| 场景 | 处理方式 |
|---|---|
| 未找到任何法条 | `status = "not_found"` |
| 找到原则性规定但无具体条款 | `status = "partial"` |
| 法条已修订/废止 | `status = "uncertain"`，标注有效版本 |
| 不同法域规定冲突 | `status = "uncertain"`，列出差异 |
| 多跳中某跳为空 | 标注该跳缺失，`status = "partial"` |
