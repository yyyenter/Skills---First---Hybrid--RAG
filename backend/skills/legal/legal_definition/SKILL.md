---

## 目标

找到法律概念的准确定义、构成要件和适用范围。返回必须包含定义来源（哪部法律、哪一条）、定义原文、构成要件分解。

## 路由决策（必须先执行）

**规则**：分析用户问题，先输出路由决策 JSON，再调用任何工具。

| 优先级 | 条件 | 方法 |
|---|---|---|
| P0 | 含"和...有什么区别"、"A和B有什么不同"、"对比" | Comparison Hop（概念对比） |
| P1 | 含"构成要件"、"需要什么条件"、"怎么成立" | Legal Element Hop（要件拆解） |
| P2 | 含"民法典和合同法"、"以前"、"现在" | Temporal Chain Hop（概念演变） |
| P3 | 以上都不匹配 | 单跳 |

```json
{"method":"Comparison Hop|Legal Element Hop|Temporal Chain Hop|single","reason":"一句话说明","sub_queries":[{"step":1,"query":"...","tool":"kb_search|kb_metadata_filter"}]}
```

### 术语规范化映射表

| 口语化表达 | 专业术语 | 映射原因 |
|---|---|---|
| "担心对方不付钱" | "不安抗辩权" | 民法典第527条 |
| "让对方先履行" | "先履行抗辩权" | 民法典第526条 |
| "一起履行" | "同时履行抗辩权" | 民法典第525条 |
| "帮人卖货" | "行纪合同 / 委托合同 / 居间合同" | 需区分3种不同合同 |
| "代理" | "委托代理 / 法定代理 / 表见代理 / 无权代理" | 需区分4种代理类型 |
| "公司管事的" | "法定代表人 / 董事 / 经理 / 实际控制人" | 需区分不同主体 |

### 近似概念辨析表

**用户可能混淆近似概念，需要辨析：**

| 用户表达 | 可能指A | 可能指B | 辨析要点 |
|---|---|---|---|
| "抗辩权" | 同时履行抗辩权 | 不安抗辩权 | 行使条件不同 |
| "代理" | 委托代理 | 表见代理 | 是否有代理权 |
| "违约" | 实际违约 | 预期违约 | 违约时间不同 |
| "解除" | 法定解除 | 约定解除 | 解除权来源不同 |
| "赔偿" | 损害赔偿 | 违约金 | 计算方式不同 |
| "合同不生效" | 无效 | 可撤销 | 法律后果不同 |

### 意图挖掘检查清单

1. **用户要的是定义，还是要构成要件？**
   - "什么是不安抗辩权" → 定义
   - "不安抗辩权需要什么条件" → 构成要件，可能需要 Legal Element Hop

2. **用户是否混淆了近似概念？**
   - "抗辩权有哪些" → 需要 Comparison Hop，对比不同类型抗辩权
   - "代理和代表有什么区别" → 需要 Comparison Hop

3. **用户是否隐含了新旧法对比？**
   - "民法典中的表见代理和合同法一样吗" → Temporal Chain Hop

4. **用户是否把多个概念混在一起？**
   - "抗辩权、抵销权、留置权有什么区别" → **三个概念对比，需要多跳**

## 多跳检索判断

### 单跳即可的场景

- 用户问的是单一概念的定义
- 问题不涉及对比、构成要件分析
- **示例**："什么是不安抗辩权？"

### 需要多跳的场景

#### 场景A：Comparison Hop（概念对比）
**适用**：用户需要对比两个或多个近似概念

**示例**：
```
问题："不安抗辩权和同时履行抗辩权有什么区别？"
分析：需要分别检索两个概念，再对比

多跳步骤：
  跳1：检索不安抗辩权
    kb_search(query="不安抗辩权 定义 构成要件 民法典", top_k=2)
    → 定义A、要件A
    
  跳2：检索同时履行抗辩权
    kb_search(query="同时履行抗辩权 定义 构成要件 民法典", top_k=2)
    → 定义B、要件B
    
  融合：对比定义A和定义B的差异
```

#### 场景B：Legal Element Hop（构成要件分析）
**适用**：用户需要了解概念的构成要件

**示例**：
```
问题："表见代理的构成要件是什么？"
分析：需要拆解构成要件

多跳步骤：
  跳1：检索表见代理的定义
    kb_search(query="表见代理 定义 民法典", top_k=2)
    → 定义：无权代理 + 权利外观 + 善意相对人
    
  跳2：检索各构成要件的具体内容
    kb_search(query="表见代理 权利外观 认定标准", top_k=2)
    kb_search(query="表见代理 善意相对人 认定", top_k=2)
    
  融合：按要件组织答案
```

#### 场景C：Temporal Chain Hop（概念演变）
**适用**：用户需要了解概念在新旧法中的变化

**示例**：
```
问题："民法典中的'合同'和合同法中的定义一样吗？"
分析：需要对比新旧法中的定义

多跳步骤：
  跳1：检索合同法中的定义
    kb_search(query="合同法 合同 定义", top_k=2)
    
  跳2：检索民法典中的定义
    kb_search(query="民法典 合同 定义 第四百六十四条", top_k=2)
    
  融合：对比新旧法定义差异
```

## 可用工具

| 工具名称 | 用途 | 必选参数 | 可选参数 |
|---|---|---|---|
| `kb_search` | 混合检索 | `query` | `top_k`, `path_filter` |
| `kb_metadata_filter` | 带元数据过滤的检索 | `query`, `filters` | `top_k` |
| `kb_open_chunk` | 打开指定 chunk 的完整内容 | `parent_id` | 无 |

---

## 检索变量说明

| 变量 | 含义 | 默认值 | 可选值 |
|---|---|---|---|
| `definition_source` | 定义来源类型 | `"statute"` | `"statute"`, `"judicial_interpretation"` |
| `jurisdiction` | 法域 | `"China"` | `"China"`, `"USA"`, `"EU"` |

---

## 执行步骤

1. **标准化术语**
   - 用术语映射表转换口语化表达
   - 辨析近似概念

2. **判断是否需要多跳**
   - 单一概念定义 → 单跳
   - 概念对比 → Comparison Hop
   - 构成要件 → Legal Element Hop
   - 新旧法对比 → Temporal Chain Hop

3. **执行检索**

4. **精确定位**
   - `kb_open_chunk` 打开定义条款
   - 提取：定义原文、构成要件、适用条件

## 推荐命令

### 单跳：定义检索

```python
kb_search(
    query="不安抗辩权 定义 构成要件",
    path_filter="knowledge/legal/statutes/",
    top_k=5
)
```

### 多跳：概念对比

```python
# 跳1：概念A
def_A = kb_search(query="不安抗辩权 定义 构成要件", top_k=2)

# 跳2：概念B
def_B = kb_search(query="同时履行抗辩权 定义 构成要件", top_k=2)
```

### 多跳：构成要件分析

```python
# 跳1：定义
definition = kb_search(query="表见代理 定义", top_k=2)

# 跳2：要件1
element1 = kb_search(query="表见代理 权利外观", top_k=2)

# 跳3：要件2
element2 = kb_search(query="表见代理 善意相对人", top_k=2)
```

## 失败处理

| 场景 | 处理方式 |
|---|---|
| 未找到定义 | `status = "not_found"` |
| 找到原则性提及但无定义 | `status = "partial"` |
| 不同来源定义冲突 | `status = "uncertain"`，列出差异 |

## 特别约束

- 定义引用必须精确到法条编号
- 构成要件必须分条列出
- 禁止用通俗解释替代法条定义
