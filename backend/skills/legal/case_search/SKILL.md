## 目标

找到与问题相关的判例、裁判规则和法院观点，回答"法院怎么判"类问题。返回必须包含案例名称、法院层级、裁判要点和适用法条。

## 路由决策（必须先执行）

**规则**：分析用户问题，先输出路由决策 JSON，再调用任何工具。

| 优先级 | 条件 | 方法 |
|---|---|---|
| P0 | 含"为什么"、"理由是什么"、"法院为什么这么判" | Bridge Entity Hop（判例→法条→理由） |
| P1 | 含"上海和北京"、"哪个更"、"对比"、"尺度一样吗" | Comparison Hop（地域/层级比较） |
| P2 | 含"以前怎么判"、"现在怎么判"、"判例被推翻" | Temporal Chain Hop（裁判规则演变） |
| P3 | 含"需要什么条件"、"构成"、"怎么认定" | Legal Element Hop（要件分析） |
| P4 | 以上都不匹配 | 单跳（One Good Case） |

```json
{"method":"Bridge Entity Hop|Comparison Hop|Temporal Chain Hop|Legal Element Hop|single","reason":"一句话说明","sub_queries":[{"step":1,"query":"...","tool":"kb_search|kb_metadata_filter"}]}
```

## 适用场景与触发条件

| 场景 | 示例 |
|---|---|
| 类似案例检索 | "有没有跳单的判例？" |
| 裁判规则查询 | "法院对除斥期间怎么认定？" |
| 指导性案例查询 | "最高法关于合同解除的指导案例有哪些？" |
| 比较型查询 | "上海和北京对竞业限制的裁判尺度一样吗？" |
| 因果关系查询 | "为什么跳单不构成违约？法院的理由是什么？" |

## 查询重写与意图挖掘

### 术语规范化映射表

| 口语化表达 | 专业术语 | 映射原因 |
|---|---|---|
| "法院怎么判" / "法官怎么说" | "裁判规则 / 裁判要点 / 法院观点 / 判决理由" | 法律检索的核心对象是裁判规则，不是判决结果 |
| "类似的案子" | "类似案例 / 类案 / 同案同判 / 参照适用" | 类案检索是法律检索的核心方法 |
| "最高法的案例" | "指导性案例 / 公报案例 / 典型案例 / 最高法案例" | 需区分不同层级的判例效力 |
| "合同不生效" | "合同无效 / 合同可撤销 / 合同解除 / 合同终止" | 法律后果不同，检索策略不同 |
| "赔钱" | "违约责任 / 损害赔偿 / 违约金 / 赔偿金" | 需区分不同责任形式 |
| "中介费" | "居间报酬 / 佣金 / 中介服务费" | 合同类型不同，法律适用不同 |
| "房子没交付" | "迟延履行 / 根本违约 / 合同目的不能实现" | 需映射到法律构成要件 |
| "公司换老板" | "股权变更 / 法定代表人变更 / 公司控制权转移" | 需映射到法律事实 |

### 意图挖掘检查清单

**执行每次检索前，先回答以下问题：**

1. **用户要的是判例本身，还是要判例背后的法条？**
   - "有没有跳单的案例" → 要判例
   - "跳单为什么不构成违约的法律依据是什么" → 判例 + 法条，可能需要 Bridge Entity Hop

2. **用户是否隐含了比较需求？**
   - "上海法院怎么判" → 地域限定，单跳
   - "上海和北京哪个更严格" → **比较型问题，需要 Comparison Hop**

3. **用户是否隐含了因果/理由追问？**
   - "跳单为什么不构成违约" → 需要 Bridge Entity Hop（跳单事实 → 法律要件 → 不构成违约的理由）
   - "法院为什么支持解除合同" → 需要 Bridge Entity Hop（解除事实 → 解除权要件 → 支持理由）

4. **用户是否把多个法律问题混在一起？**
   - "合同无效了，钱能不能退，中介费怎么办" → **三个独立法律问题，必须拆成3个子查询**

5. **用户是否混淆了法律概念？**
   - "合同不生效" → 需判断用户说的是"无效"、"可撤销"、"解除"还是"终止"，每个对应不同检索策略
   - "违约责任" → 需判断是"继续履行"、"损害赔偿"还是"违约金"，检索方向不同

### 查询重写策略

**策略1：法律要件拆解重写**
```
原始："跳单为什么不构成违约？"
  ↓ 拆解法律要件
要件1：跳单事实的认定标准
要件2：居间合同的违约构成要件
要件3：不构成违约的抗辩理由
  ↓ 重写
"跳单 居间合同 违约构成要件"
"居间合同 买方权利 不构成违约"
"最高法 跳单 裁判规则"
```

**策略2：法条编号引导**
```
原始："合同解除的条件"
重写："民法典第563条 合同解除 条件 判例"
重写："民法典第563条 解除权 除斥期间 判例"
```

**策略3：法院层级限定**
```
原始："关于格式条款的判例"
重写："指导性案例 格式条款 民法典第496条"
重写："公报案例 格式条款 提示说明义务"
```

**策略4：时间范围限定**
```
原始："最近的劳动争议判例"
重写："劳动争议 2024 判例 典型案例"
```

## 多跳检索判断

### 单跳即可的场景（直接执行 kb_search）

- 问题只有一个明确的法律关系和事实特征
- 用户只需要某个具体问题的判例列表
- 问题不涉及比较、因果、法条推导
- **示例**："有没有关于跳单的判例？"

### 需要多跳的场景（必须拆分执行）

#### 场景A：Bridge Entity Hop（桥接实体跳 / One Good Case Method）
**适用**：问题需要先找到关键法条/判例，再用其作为桥梁找更多相关判例

**判断标准**：
- 问题形式："关于X的判例"，但 X 是一个法律原则/法条
- 问题需要法条 → 判例的推导："民法典第563条有哪些判例？"
- 问题需要判例 → 法条的回溯："这个判例引用了什么法条？"

**One Good Case Method 标准流程**：
```
问题："关于合同解除权的判例"

多跳步骤：
  跳1：找到一个"好案例"
    kb_search(query="合同解除权 指导性案例", top_k=3)
    → 找到"指导案例1号"（或其他高层级判例）
    
  跳2：提取该判例引用的核心法条
    kb_open_chunk(parent_id="contract_termination_case::指导案例 1 号")
    → 提取法条："民法典第496条、第563条"
    
  跳3：用法条编号检索更多判例
    kb_search(query="民法典第563条 合同解除 判例", top_k=5)
    → 找到所有引用该法条的判例
    
  跳4（可选）：用判例中的关键词扩展检索
    kb_search(query="除斥期间 合同解除 裁判规则", top_k=3)
```

**执行代码**：
```python
# 跳1：找一个好案例
good_case = kb_search(query="合同解除权 指导性案例", top_k=3)
case_parent_id = good_case[0].parent_id  # 提取 parent_id

# 跳2：读取完整案例，提取法条
full_case = kb_open_chunk(parent_id=case_parent_id)
law_articles = extract_law_articles(full_case)  # 如 "民法典第563条"

# 跳3：用法条找更多案例
more_cases = kb_search(query=f"{law_articles} 判例", top_k=5)
```

#### 场景B：Comparison Hop（比较跳 / 地域/层级比较）
**适用**：问题涉及不同地域、不同时期、不同层级的裁判规则比较

**判断标准**：
- 问题含"哪个更严格"、"上海和北京"、"以前和现在"、"高院和最高法"
- 问题涉及裁判尺度的统一性/差异性

**示例**：
```
问题："上海和北京对竞业限制的裁判尺度一样吗？"
分析：这是比较型问题，必须分别检索两地判例

多跳步骤：
  跳1：检索上海地区的竞业限制判例
    kb_search(query="竞业限制 上海 判例", top_k=3)
    → 收集上海裁判规则A
    
  跳2：检索北京地区的竞业限制判例
    kb_search(query="竞业限制 北京 判例", top_k=3)
    → 收集北京裁判规则B
    
  跳3：检索最高法对竞业限制的统一规则
    kb_search(query="竞业限制 最高法 指导性案例", top_k=3)
    → 收集最高法统一规则C
    
  融合：对比A、B、C的差异，分析是否一致
```

**执行代码**：
```python
# 跳1：检索地域A
rules_A = kb_search(query="竞业限制 上海 判例", top_k=3)

# 跳2：检索地域B
rules_B = kb_search(query="竞业限制 北京 判例", top_k=3)

# 跳3：检索最高法统一规则
rules_supreme = kb_search(query="竞业限制 最高法 指导性案例", top_k=3)
```

#### 场景C：Temporal Chain Hop（时序链跳 / 裁判规则演变）
**适用**：问题涉及裁判规则的时间演变、新旧法适用、判例 overturn

**判断标准**：
- 问题含"以前怎么判"、"现在怎么判"、"新法出来后"、"判例被推翻"
- 问题涉及法律溯及力或判例效力变化

**示例**：
```
问题："民法典实施后，关于合同解除的裁判规则有什么变化？"
分析：这是时序问题，需要对比民法典前后的裁判规则

多跳步骤：
  跳1：检索民法典实施前（2020年前）的裁判规则
    kb_search(query="合同解除 合同法 判例 year_min=2015 year_max=2020", top_k=3)
    → 收集旧法规则A
    
  跳2：检索民法典实施后（2021年后）的裁判规则
    kb_search(query="合同解除 民法典第563条 判例 year_min=2021", top_k=3)
    → 收集新法规则B
    
  跳3：检索最高法关于民法典适用的司法解释
    kb_search(query="民法典 时间效力 司法解释 合同", top_k=2)
    → 收集司法解释C
    
  融合：对比A、B、C，分析规则变化
```

#### 场景D：Legal Element Hop（法律要件跳）
**适用**：问题涉及法律构成要件的分析，需要拆解要件逐步验证

**判断标准**：
- 问题形式："X是否构成Y？"、"X为什么不构成Y？"
- 问题涉及法律构成要件（主体、客体、主观方面、客观方面）

**示例**：
```
问题："为什么跳单不构成违约？"
分析：需要拆解违约的构成要件，逐一验证

多跳步骤：
  跳1：检索违约的构成要件
    kb_search(query="居间合同 违约 构成要件", top_k=2)
    → 要件1：合同成立；要件2：违约行为；要件3：因果关系；要件4：损害结果
    
  跳2：检索跳单行为的事实认定
    kb_search(query="跳单 事实认定 判例", top_k=2)
    → 跳单行为是否满足"违约行为"要件
    
  跳3：检索不构成违约的抗辩理由
    kb_search(query="跳单 不构成违约 抗辩 判例", top_k=2)
    → 买方通过其他正当途径获得信息 → 不满足"违约行为"要件
    
  融合：按构成要件组织答案
```

### 多跳执行通用流程

```
Step 1: 判断是否需要多跳
  └─→ 如果单跳即可 → 直接执行 kb_search
  └─→ 如果需要多跳 → 继续

Step 2: 选择多跳方法
  └─→ 含"法条→判例"或"判例→法条" → One Good Case (Bridge Entity Hop)
  └─→ 含"比较/哪个更" → Comparison Hop
  └─→ 含"以前/现在/变化" → Temporal Chain Hop
  └─→ 含"为什么/构成要件" → Legal Element Hop
  └─→ 混合类型 → 组合使用

Step 3: 拆分子查询
  └─→ 每个子查询对应一个法律要件、一个地域、一个时间点
  └─→ 子查询之间必须有逻辑依赖关系
  └─→ 每个子查询调用一次 kb_search 或 kb_metadata_filter

Step 4: 逐步执行
  └─→ 跳1：执行第一个子查询
  └─→ 提取关键实体（法条编号、判例名称、裁判要点）
  └─→ 跳2：用提取的实体构建第二个查询
  └─→ ...
  └─→ 跳N：最终查询

Step 5: 融合所有跳的结果
  └─→ 按法律逻辑组织答案
  └─→ 标注每跳的查询和来源
  └─→ 如果某跳返回空，标注并降级结论
```

## 可用工具

| 工具名称 | 用途 | 必选参数 | 可选参数 |
|---|---|---|---|
| `kb_search` | 混合检索（向量 + BM25） | `query` | `top_k`, `path_filter` |
| `kb_metadata_filter` | 带元数据过滤的检索 | `query`, `filters` | `top_k` |
| `kb_open_chunk` | 打开指定 chunk 的完整内容 | `parent_id` | 无 |

---

## 检索变量说明

| 变量 | 含义 | 默认值 | 可选值 |
|---|---|---|---|
| `court_level` | 法院层级 | 无优先级 | `"supreme"`, `"high"`, `"intermediate"`, `"basic"` |
| `year_min` | 最早年份 | 2020 | 任意整数 |
| `case_type` | 案件类型 | 无 | `"civil"`, `"criminal"`, `"administrative"`, `"contract"` |
| `legal_domain` | 法律领域 | 无 | `"contract"`, `"tort"`, `"property"`, `"ip"`, `"labor"` |

## 执行步骤

### 单跳执行流程

1. **提取检索要素**
   - 核心法律关系（如"居间合同"、"预售合同"）
   - 关键事实特征（如"跳单"、"除斥期间"）
   - 法院层级偏好（未指定则优先最高法）

2. **查询重写**
   - 用"术语规范化映射表"把口语化转成法律术语
   - 用"查询重写策略"扩展同义词和限定条件

3. **首次检索（One Good Case）**
   - 用 `kb_metadata_filter`，filters 包含 `document_type=case`
   - 优先过滤 `court_level=supreme`
   - 优先过滤 `year_min=2020`（近5年优先）

4. **结果筛选**
   - 从结果中选择最相关的 1-3 个判例
   - 用 `kb_open_chunk` 读取完整裁判要点

### 多跳执行流程

1. **判断多跳类型**（见"多跳检索判断"章节）
2. **拆分子查询**，每个子查询对应一个法律要件/地域/时间点
3. **逐跳执行**，每跳后提取关键实体（法条编号、判例名称）
4. **融合结果**，按法律逻辑组织

## 推荐命令

### 单跳：One Good Case

```python
kb_metadata_filter(
    query="商品房预售合同 解除权 除斥期间",
    filters={
        "document_type": "case",
        "court_level": "supreme",
        "year_min": 2020
    },
    top_k=8
)
```

### 多跳：One Good Case Method（法条→判例）

```python
# 跳1：找一个好案例
good_case = kb_search(query="合同解除权 指导性案例", top_k=3)
case_id = good_case[0].parent_id

# 跳2：读取完整案例，提取法条
full_case = kb_open_chunk(parent_id=case_id)
articles = extract_law_articles(full_case)  # 如 "民法典第563条"

# 跳3：用法条找更多案例
more_cases = kb_search(query=f"{articles} 判例", top_k=5)
```

### 多跳：Comparison Hop（地域比较）

```python
# 跳1：检索地域A
rules_A = kb_search(query="竞业限制 上海 判例", top_k=3)

# 跳2：检索地域B
rules_B = kb_search(query="竞业限制 北京 判例", top_k=3)

# 跳3：检索最高法统一规则
rules_supreme = kb_search(query="竞业限制 最高法 指导性案例", top_k=3)
```

### 多跳：Legal Element Hop（要件分析）

```python
# 跳1：检索构成要件
elements = kb_search(query="居间合同 违约 构成要件", top_k=2)

# 跳2：检索事实认定
facts = kb_search(query="跳单 事实认定 判例", top_k=2)

# 跳3：检索抗辩理由
defense = kb_search(query="跳单 不构成违约 抗辩", top_k=2)
```

## 结果筛选规则

- 优先**指导性案例**、**公报案例**、**最高法典型案例**
- 同一法律问题多个判例时，优先**最新判例**
- 注意判例是否被后续判例**推翻或限缩**
- 对比判例时要标注法院层级差异
- 不要只用一条判例下结论，至少引用 2-3 个判例

---

## 失败处理

| 场景 | 处理方式 |
|---|---|
| 未找到任何判例 | `status = "not_found"`，说明"未找到相关判例" |
| 仅找到原则性论述 | `status = "partial"`，说明"找到裁判思路，但缺少具体判例支撑" |
| 判例之间存在冲突 | `status = "uncertain"`，列出冲突判例并标注法院层级 |
| 判例已被推翻 | `status = "uncertain"`，标注最新裁判规则 |
| 多跳中某跳为空 | 标注该跳缺失，用其他跳的结果补充，`status = "partial"` |
