## 目标

找到最新临床指南中的诊疗建议和推荐等级，回答"应该怎么治"类问题。返回必须包含指南来源、推荐等级、具体建议内容。

## 路由决策（必须先执行）

**规则**：分析用户问题，先输出路由决策 JSON，再调用任何工具。

| 优先级 | 条件 | 方法 |
|---|---|---|
| P0 | 含"中国和美国"、"ADA和"、"不同指南" | Comparison Hop（指南对比） |
| P1 | 含"为什么推荐"、"证据是什么" | Bridge Entity Hop（指南→循证证据） |
| P2 | 含"老年人"、"孕妇"、"肾不好"、"肝功能不全" | Special Population Hop（通用→特殊人群调整） |
| P3 | 以上都不匹配 | 单跳 |

```json
{"method":"Comparison Hop|Bridge Entity Hop|Special Population Hop|single","reason":"一句话说明","sub_queries":[{"step":1,"query":"...","tool":"kb_search|kb_metadata_filter"}]}
```

### 术语规范化映射表

| 口语化表达 | 专业术语 | 映射原因 |
|---|---|---|
| "血糖高" | "2型糖尿病 / T2DM / 高血糖" | 需具体化到疾病诊断 |
| "血压高" | "高血压 / 原发性高血压" | 需区分原发/继发 |
| "胆固醇高" | "高脂血症 / 高胆固醇血症 / LDL-C升高" | 需具体化到血脂指标 |
| "吃多少药" | "剂量 / 起始剂量 / 维持剂量 / 最大剂量" | 需对应具体用药方案 |
| "什么时候吃药" | "给药时间 / 餐前 / 餐后 / 睡前" | 影响药物吸收 |
| "多久复查" | "随访频率 / 监测周期 / 复诊间隔" | 指南中的管理策略 |
| "能不能治好" | "缓解 / 控制 / 达标 / 预后" | 需管理患者预期 |

| "老人/小孩能吃吗" | "特殊人群 / 老年 / 儿童 / 妊娠 / 肝肾功能不全" | 需检索特殊人群用药指南 |
| "吃药多久见效" | "起效时间 / 疗效评估时间 / HbA1c评估周期" | 指南中的时间节点 |

### 意图挖掘检查清单

1. **用户要的是指南推荐，还是要证据等级？**
   - "应该怎么治" → 指南推荐
   - "有什么证据支持" → 需要 Bridge Entity Hop（指南→循证文献）

2. **用户是否隐含了比较需求？**
   - "中国和美国指南有什么不同" → **Comparison Hop**
   - "指南和临床经验哪个更可靠" → 需要指南 vs 证据的对比

3. **用户是否隐含了特殊人群？**
   - "老年人怎么治" → 需限定特殊人群
   - "肾不好能吃吗" → 需检索肾功能不全的用药调整

4. **用户是否把多个问题混在一起？**
   - "糖尿病怎么治，吃什么药，多久复查" → **三个问题，需拆分**

### 查询重写策略

**策略1：疾病+干预+指南**
```
原始："糖尿病怎么治"
重写："2型糖尿病 一线治疗 指南推荐 2024"
```

**策略2：特殊人群限定**
```
原始："肾不好能吃二甲双胍吗"
重写："二甲双胍 肾功能不全 eGFR 指南 剂量调整"
```

**策略3：指南对比**
```
原始："中国和美国指南有什么不同"
重写1："2型糖尿病 指南 中国 2024"
重写2："T2DM guideline ADA 2024"
```

## 多跳检索判断

### 单跳即可的场景

- 用户问的是单一疾病的指南推荐
- 不涉及比较、特殊人群、证据追溯
- **示例**："2型糖尿病的一线治疗是什么？"

### 需要多跳的场景

#### 场景A：Bridge Entity Hop（指南→循证证据）
**适用**：用户需要了解指南推荐背后的证据

**示例**：
```
问题："为什么二甲双胍是一线推荐？"
分析：需要指南推荐 + 循证证据

多跳步骤：
  跳1：检索指南推荐
    kb_search(query="二甲双胍 一线推荐 指南", top_k=2)
    → 指南：强推荐，A级证据
    
  跳2：检索循证证据
    kb_search(query="metformin T2DM systematic review RCT", top_k=2)
    → 证据：UKPDS研究，降低心血管风险
    
  融合：指南推荐 + 证据 = 完整答案
```

#### 场景B：Comparison Hop（指南对比）
**适用**：用户需要对比不同指南的推荐

**示例**：
```
问题："中国指南和ADA指南对血糖目标有什么不同？"
分析：需要分别检索两个指南，再对比

多跳步骤：
  跳1：检索中国指南
    kb_search(query="2型糖尿病 血糖目标 中国指南 2024", top_k=2)
    → 中国指南：HbA1c < 7.0%
    
  跳2：检索ADA指南
    kb_search(query="T2DM glycemic target ADA guideline 2024", top_k=2)
    → ADA指南：HbA1c < 7.0%（大部分成人）
    
  融合：对比差异
```

#### 场景C：Special Population Hop（特殊人群检索）
**适用**：用户问题涉及特殊人群，需要先检索通用指南，再检索特殊人群调整

**示例**：
```
问题："肾功能不全的糖尿病患者怎么治？"
分析：需要先检索通用指南，再检索肾功能不全的调整

多跳步骤：
  跳1：检索通用指南
    kb_search(query="2型糖尿病 治疗 指南", top_k=2)
    → 通用推荐：二甲双胍一线
    
  跳2：检索肾功能不全调整
    kb_search(query="二甲双胍 肾功能不全 eGFR 指南 剂量调整", top_k=2)
    → eGFR < 45：减量；eGFR < 30：禁用
    
  融合：通用推荐 + 特殊调整
```

## 可用工具

| 工具名称 | 用途 | 必选参数 | 可选参数 |
|---|---|---|---|
| `kb_search` | 混合检索 | `query` | `top_k`, `path_filter` |
| `kb_metadata_filter` | 带元数据过滤的检索 | `query`, `filters` | `top_k` |
| `kb_open_chunk` | 打开指定 chunk 的完整内容 | `parent_id` | 无 |

---

## 检索变量说明

| 变量 | 含义 | 默认值 | 可选值 |
|---|---|---|---|
| `organization` | 发布机构 | 无 | `"中华医学会"`, `"NCCN"`, `"WHO"`, `"ADA"` |
| `year_min` | 最早年份 | 2020 | 任意整数 |
| `evidence_level` | 证据等级 | 无 | `"Level I"`, `"Level II"`, `"Level III"` |
| `disease` | 疾病名称 | 自动提取 | 如 `"T2DM"`, `"hypertension"` |

---

## 执行步骤

1. **疾病名称标准化**
   - 用疾病名称标准化表转换

2. **查询重写**
   - 用术语映射表转换
   - 识别特殊人群

3. **判断是否需要多跳**
   - 需要证据追溯 → Bridge Entity Hop
   - 需要指南对比 → Comparison Hop
   - 需要特殊人群 → Special Population Hop

4. **执行检索**

5. **推荐等级提取**

## 推荐命令

### 单跳：按疾病检索指南

```python
kb_metadata_filter(
    query="2型糖尿病 一线治疗 二甲双胍",
    filters={
        "document_type": "guideline",
        "organization": "中华医学会",
        "year_min": 2020
    },
    top_k=5
)
```

### 多跳：指南→证据

```python
# 跳1：指南推荐
guideline = kb_search(query="二甲双胍 一线推荐 指南", top_k=2)

# 跳2：循证证据
evidence = kb_search(query="metformin T2DM systematic review", top_k=2)
```

### 多跳：指南对比

```python
# 跳1：中国指南
cn_guide = kb_search(query="2型糖尿病 血糖目标 中国指南", top_k=2)

# 跳2：ADA指南
ada_guide = kb_search(query="T2DM glycemic target ADA", top_k=2)
```

## 失败处理

| 场景 | 处理方式 |
|---|---|
| 未找到指南 | `status = "not_found"` |
| 找到指南但无具体推荐 | `status = "partial"` |
| 指南间推荐冲突 | `status = "uncertain"`，列出冲突 |

## 特别约束

- 必须标注指南来源和年份
- 必须标注推荐等级和证据等级
- 禁止用个人经验替代指南推荐
