## 目标

找到药品的法定信息（说明书、药典）和指南推荐。返回必须包含药品名称、适应症、用法用量、禁忌症、主要不良反应。

## 路由决策（必须先执行）

**规则**：分析用户问题，先输出路由决策 JSON，再调用任何工具。

| 优先级 | 条件 | 方法 |
|---|---|---|
| P0 | 含"哪个更好"、"A和B"、"比较"、"选择" | Comparison Hop（分别检索A/B→对比） |
| P1 | 含"为什么首选"、"指南推荐" | Bridge Entity Hop（药物→指南推荐等级） |
| P2 | 含"肾不好"、"孕妇"、"老年人"、"肝功能" | Special Population Hop（通用→特殊人群剂量调整） |
| P3 | 含"相互作用"、"一起吃" | 单跳（联合用药检索） |
| P4 | 以上都不匹配 | 单跳 |

```json
{"method":"Comparison Hop|Bridge Entity Hop|Special Population Hop|single","reason":"一句话说明","sub_queries":[{"step":1,"query":"...","tool":"kb_search|kb_metadata_filter"}]}
```

### 术语规范化映射表

| 口语化表达 | 专业术语 | 映射原因 |
|---|---|---|

| "降压药" | "ACEI / ARB / CCB / 利尿剂 / β受体阻滞剂" | 必须具体化到药物类别 |
| "降脂药" | "他汀类 / 贝特类 / PCSK9抑制剂" | 需具体化 |
| "怎么吃" | "用法用量 / 起始剂量 / 维持剂量 / 给药频次" | 需精确到mg和频次 |
| "伤肾" | "肾毒性 / 肾功能不全 / eGFR下降" | 需精确到肾功指标 |

### 药物名称标准化

| 用户表达 | 通用名 | 商品名 |
|---|---|---|
| "格华止" | "二甲双胍" | Glucophage |
| "拜糖平" | "阿卡波糖" | Glucobay |

### 意图挖掘检查清单

1. **用户要的是药品说明书，还是要指南推荐？**
   - "二甲双胍怎么吃" → 说明书信息
   - "二甲双胍是不是一线推荐" → 指南推荐，需要 Bridge Entity Hop

2. **用户是否隐含了比较需求？**
   - "二甲双胍和胰岛素哪个好" → **Comparison Hop**
   - "哪种降糖药不伤肾" → 需要多药对比

3. **用户是否隐含了特殊人群？**
   - "肾不好能吃吗" → 需要检索肾功能不全的剂量调整
   - "孕妇能吃吗" → 需要检索妊娠分级

4. **用户是否把多个问题混在一起？**
   - "二甲双胍怎么吃，有什么副作用，肾不好能吃吗" → **三个问题，需拆分**

### 查询重写策略

**策略1：通用名优先**
```
原始："格华止怎么吃"
重写："二甲双胍 用法用量 说明书"
```

**策略2：特殊人群限定**
```
原始："肾不好能吃二甲双胍吗"
重写："二甲双胍 肾功能不全 eGFR 剂量调整 禁忌"
```

**策略3：药物对比**
```
原始："二甲双胍和格列美脲哪个好"
重写1："二甲双胍 疗效 安全性 指南"
重写2："格列美脲 疗效 安全性 指南"
```

## 多跳检索判断

### 单跳即可的场景

- 用户问的是单一药物的单一信息
- 不涉及比较、特殊人群
- **示例**："二甲双胍的用法用量是什么？"

### 需要多跳的场景

#### 场景A：Bridge Entity Hop（药物→指南推荐）
**适用**：用户需要了解药物在指南中的地位

**示例**：
```
问题："二甲双胍为什么是首选？"
分析：需要药物信息 + 指南推荐

多跳步骤：
  跳1：检索药物说明书
    kb_search(query="二甲双胍 说明书 适应症", top_k=2)
    → 说明书：T2DM一线用药
    
  跳2：检索指南推荐
    kb_search(query="二甲双胍 一线推荐 指南 证据等级", top_k=2)
    → 指南：强推荐，A级证据
    
  融合：说明书 + 指南 = 完整答案
```

#### 场景B：Comparison Hop（药物对比）
**适用**：用户需要对比两种药物

**示例**：
```
问题："二甲双胍和格列美脲哪个更适合老年人？"
分析：需要分别检索两种药物在老年人中的信息

多跳步骤：
  跳1：检索二甲双胍老年用药
    kb_search(query="二甲双胍 老年 肾功能 安全性", top_k=2)
    → 结果A
    
  跳2：检索格列美脲老年用药
    kb_search(query="格列美脲 老年 低血糖 安全性", top_k=2)
    → 结果B
    
  融合：对比结果A和结果B
```

#### 场景C：Special Population Hop（特殊人群用药）
**适用**：用户问题涉及特殊人群

**示例**：
```
问题："肾功能不全的糖尿病患者能吃什么药？"
分析：需要先检索可用药物，再检索每种药物在肾功能不全时的剂量调整

多跳步骤：
  跳1：检索糖尿病药物列表
    kb_search(query="2型糖尿病 药物治疗 指南", top_k=2)
    → 药物列表：二甲双胍、SGLT2抑制剂、DPP-4抑制剂等
    
  跳2：检索各药物在肾功能不全时的调整
    kb_search(query="二甲双胍 肾功能不全 eGFR 剂量调整", top_k=2)
    kb_search(query="SGLT2抑制剂 肾功能不全 eGFR 禁忌", top_k=2)
    
  融合：整理各药物的肾功能调整方案
```

## 可用工具

| 工具名称 | 用途 | 必选参数 | 可选参数 |
|---|---|---|---|
| `kb_search` | 混合检索 | `query` | `top_k`, `path_filter` |
| `kb_metadata_filter` | 带元数据过滤的检索 | `query`, `filters` | `top_k` |
| `kb_open_chunk` | 打开指定 chunk 的完整内容 | `parent_id` | 无 |

---

## 检索变量说明

| 变量 | 含义 | 默认值 | 可选值 |
|---|---|---|---|
| `drug_name` | 药品通用名 | 自动提取 | 如 `"metformin"`, `"二甲双胍"` |
| `drug_class` | 药物类别 | 无 | 如 `"biguanide"`, `"ACEI"` |
| `population` | 特殊人群 | 无 | `"pregnant"`, `"elderly"`, `"renal_impairment"` |

---

## 执行步骤

1. **标准化药名**
   - 商品名→通用名
   - 俗名→通用名

2. **查询重写**
   - 用术语映射表转换
   - 识别特殊人群

3. **判断是否需要多跳**
   - 需要指南推荐 → Bridge Entity Hop
   - 需要药物对比 → Comparison Hop
   - 需要特殊人群 → Special Population Hop

4. **执行检索**

5. **信息分层提取**

## 推荐命令

### 单跳：按药品通用名检索

```python
kb_metadata_filter(
    query="二甲双胍 肾功能不全 eGFR",
    filters={
        "document_type": "drug_label",
        "drug_name": "metformin"
    },
    top_k=5
)
```

### 多跳：药物→指南

```python
# 跳1：药物信息
drug = kb_search(query="二甲双胍 说明书", top_k=2)

# 跳2：指南推荐
guide = kb_search(query="二甲双胍 指南 一线推荐", top_k=2)
```

### 多跳：药物对比

```python
# 跳1：药物A
drug_A = kb_search(query="二甲双胍 老年 安全性", top_k=2)

# 跳2：药物B
drug_B = kb_search(query="格列美脲 老年 安全性", top_k=2)
```

## 失败处理

| 场景 | 处理方式 |
|---|---|
| 未找到药品信息 | `status = "not_found"` |
| 找到药品但缺少特殊人群信息 | `status = "partial"` |

## 特别约束

- 必须标注药品通用名和商品名
- 剂量信息必须精确到 mg 和频次
- 禁忌症必须完整列出
- 禁止推荐超说明书用药
