## 目标

找到疾病的权威诊断标准、分型分期体系和检查方法。返回必须包含诊断标准来源、具体指标、排除标准。

## 路由决策（必须先执行）

**规则**：分析用户问题，先输出路由决策 JSON，再调用任何工具。

| 优先级 | 条件 | 方法 |
|---|---|---|
| P0 | 含"怎么区分"、"A和B有什么不同"、"鉴别" | Comparison Hop（分别检索A/B诊断标准→对比） |
| P1 | 含"口渴多尿血糖7.5"等具体症状+指标 | Bridge Entity Hop（症状→检查指标→诊断标准验证） |
| P2 | 含"从...到..."、"进展"、"分期" | Temporal Chain Hop（分期标准→进展时间） |
| P3 | 以上都不匹配 | 单跳 |

```json
{"method":"Comparison Hop|Bridge Entity Hop|Temporal Chain Hop|single","reason":"一句话说明","sub_queries":[{"step":1,"query":"...","tool":"kb_search|kb_metadata_filter"}]}
```

### 意图挖掘检查清单

1. **用户要的是诊断标准，还是要鉴别诊断？**
   - "怎么诊断糖尿病" → 诊断标准
   - "怎么区分1型和2型" → **鉴别诊断，需要 Comparison Hop**

2. **用户是否隐含了分期/分级需求？**
   - "糖尿病严重吗" → 需要分期信息
   - "肾病到几期了" → 需要 CKD 分期

   - "我最近口渴、多尿，是不是糖尿病" → 需要症状→诊断标准的推导

4. **用户是否问的是正常值而非诊断标准？**

## 多跳检索判断

### 单跳即可的场景

- 用户问的是单一疾病的诊断标准
- 不涉及鉴别诊断、分期
- **示例**："糖尿病的诊断标准是什么？"

### 需要多跳的场景

#### 场景A：Comparison Hop（鉴别诊断）
**适用**：用户需要区分两种疾病

**示例**：
```
问题："怎么区分1型和2型糖尿病？"
分析：需要分别检索两种疾病的诊断标准，再对比

多跳步骤：
  跳1：检索1型糖尿病诊断标准
    kb_search(query="1型糖尿病 诊断标准 抗体 GAD65", top_k=2)
    → 标准A：GAD65抗体阳性，酮症倾向
    
  跳2：检索2型糖尿病诊断标准
    kb_search(query="2型糖尿病 诊断标准 胰岛素抵抗", top_k=2)
    → 标准B：胰岛素抵抗，无抗体
    
  融合：对比标准A和标准B的差异
```

#### 场景B：Bridge Entity Hop（症状→诊断标准）
**适用**：用户描述症状，需要推导诊断标准

**示例**：
```
问题："我最近口渴、多尿，血糖7.5，是不是糖尿病？"
分析：需要症状→检查指标→诊断标准

多跳步骤：
  跳1：检索诊断标准
    kb_search(query="糖尿病 诊断标准 空腹血糖 HbA1c", top_k=2)
    → 标准：FPG ≥ 7.0 或 HbA1c ≥ 6.5
    
  跳2：验证用户指标
    kb_search(query="空腹血糖 7.5 诊断 糖尿病", top_k=2)
    → FPG 7.5 ≥ 7.0，符合诊断标准
    
  融合：诊断标准 + 用户指标 = 诊断判断
```

#### 场景C：Temporal Chain Hop（疾病进展）
**适用**：用户需要了解疾病分期和进展

**示例**：
```
问题："糖尿病肾病从微量白蛋白尿到肾衰竭要多久？"
分析：需要分期标准 + 进展时间

多跳步骤：
  跳1：检索糖尿病肾病分期
    kb_search(query="糖尿病肾病 分期 CKD eGFR 白蛋白尿", top_k=2)
    → 分期标准
    
  跳2：检索进展时间
    kb_search(query="糖尿病肾病 进展时间 微量白蛋白尿 肾衰竭", top_k=2)
    → 进展时间
    
  融合：分期 + 进展时间 = 完整答案
```

## 可用工具

| 工具名称 | 用途 | 必选参数 | 可选参数 |
|---|---|---|---|
| `kb_search` | 混合检索 | `query` | `top_k`, `path_filter` |
| `kb_metadata_filter` | 带元数据过滤的检索 | `query`, `filters` | `top_k` |
| `kb_open_chunk` | 打开指定 chunk 的完整内容 | `parent_id` | 无 |

---

## 检索变量说明

| 变量 | 含义 | 默认值 | 可选值 |
|---|---|---|---|
| `disease_name` | 疾病名称 | 自动提取 | 如 `"T2DM"`, `"hypertension"` |
| `diagnosis_standard` | 诊断标准版本 | 最新 | `"WHO_2024"`, `"ADA_2024"` |
| `classification` | 分型/分期 | 无 | `"type"`, `"stage"`, `"grade"` |

---

## 执行步骤

1. **确定疾病名称**
   - 标准化疾病名称
   - 区分原发/继发

2. **查询重写**
   - 用术语映射表转换
   - 区分诊断标准 vs 正常值

3. **判断是否需要多跳**
   - 需要鉴别诊断 → Comparison Hop
   - 需要症状推导 → Bridge Entity Hop
   - 需要分期 → Temporal Chain Hop

4. **执行检索**

5. **诊断标准提取**

## 推荐命令

### 单跳：诊断标准检索

```python
kb_metadata_filter(
    query="2型糖尿病 诊断标准 HbA1c",
    filters={
        "document_type": "guideline",
        "disease": "T2DM"
    },
    top_k=5
)
```

### 多跳：鉴别诊断

```python
# 跳1：疾病A
diag_A = kb_search(query="1型糖尿病 诊断标准 抗体", top_k=2)

# 跳2：疾病B
diag_B = kb_search(query="2型糖尿病 诊断标准 胰岛素抵抗", top_k=2)
```

### 多跳：症状→诊断

```python
# 跳1：诊断标准
standard = kb_search(query="糖尿病 诊断标准 空腹血糖", top_k=2)

# 跳2：验证指标
verification = kb_search(query="空腹血糖 7.5 糖尿病 诊断", top_k=2)
```

## 失败处理

| 场景 | 处理方式 |
|---|---|
| 未找到诊断标准 | `status = "not_found"` |
| 找到标准但缺少切点 | `status = "partial"` |
| 新旧标准差异大 | `status = "uncertain"`，列出差异 |

## 特别约束

- 诊断标准必须标注版本和来源
- 必须区分主要诊断标准和次要诊断标准
- 必须列出排除标准
- 禁止用非权威来源的诊断标准
